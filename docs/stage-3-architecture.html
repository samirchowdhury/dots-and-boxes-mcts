Stage 3 Model Architecture

Each block gets its own small interactive view: input, conv stem, residual body, policy head, and value head.

docs/stage-3-architecture.html

1. Encoder

The 4x4-dot board becomes a 7x7 spatial grid. Dots, edges, and boxes keep their natural locations.

C x 7 x 7
dot P0 edge P1 edge box

The channels are stacked over this same 7x7 coordinate system: drawn edges, owners, boxes, score context, current player, and legal moves.

2. Conv Stem

A 3x3 kernel sweeps over all 7x7 output positions. One zero-padding border lets corner and edge positions exist.

padding 1, 3x3 conv -> 64 x 7 x 7
Drawn edges0
Legal edges0
Padding cells0
Patch reads aslocal tactic

Because padding supplies zeros outside the real board, the kernel center can visit row 0, col 0 through row 6, col 6 and still produce a 7x7 feature map.

3. Residual Body

Repeated residual blocks refine the shared feature maps while keeping the tensor shape fixed.

4-6 blocks, 64 x 7 x 7
Trunk features 64 x 7 x 7

Feature maps from the convolutional stem.

|
ResidualBlock 1 64 x 7 x 7

Two padded 3x3 convolutions plus a skip connection.

|
ResidualBlock 2 64 x 7 x 7

Same operation, same shape, refined features.

|
... repeat 4-6 total

Depth adds more feature refinement without resizing the board.

|
Residual body output 64 x 7 x 7

The policy and value heads both read this shared tensor.

A residual block is still mostly convolution. The skip connection just adds the input back in, so the block can learn a correction while preserving useful features.

4. Policy Head

The policy branch converts spatial features into one score for each playable edge.

24 edge logits
Trunk output 64 x 7 x 7

Every board coordinate has learned features.

|
1x1 policy conv 2 x 7 x 7

A cheap per-cell projection into policy feature maps.

|
Flatten edge cells 98 -> 24

Keep only horizontal and vertical edge coordinates.

|
Mask + softmax 24 probabilities

Already drawn edges get removed before probabilities are used.

The policy head is still spatial until the edge-only flattening step. That is the bridge from board-shaped tensors to move ids like h:2:0.

5. Value Head

The value branch compresses all board features into one scalar for the current state.

scalar in [-1, 1]
Trunk output 64 x 7 x 7

The same shared features used by the policy head.

|
1x1 value conv 1 x 7 x 7

Compress channels while keeping board positions.

|
Flatten + MLP 49 -> 64 -> 1

Collect evidence from the whole board.

|
tanh v in [-1, 1]

Predict final outcome from the player-to-move perspective.

Target score margin / boxes
Loss MSE or Huber

Unlike the policy head, the value head intentionally destroys spatial shape near the end because it needs one board-level judgment.
